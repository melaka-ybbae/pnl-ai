# AI 재무 플랫폼 데모 시나리오

## 핵심 영업 포인트

> **"깔끔한 데이터 정리해서 분석하는 건 쉽습니다.
> 하지만 현실의 지저분한 데이터도 저희 AI가 자동으로 처리합니다."**

---

## 데모 시나리오 1: AI 스마트 파싱 (핵심!)

### 목표
- AI가 다양한 형식의 엑셀 데이터를 자동 인식하는 것을 보여줌
- 컬럼명이 다르거나, 형식이 제각각이어도 처리 가능
- 이상치/오류도 자동 감지

### 샘플 파일 위치
```
sample_data/messy_samples/
├── 매출전표_더존스타일.xlsx      # 더존 ERP 스타일
├── 매출전표_SAP스타일.xlsx       # SAP 스타일 (영문)
├── 매출전표_자체양식.xlsx        # 회사 자체 양식
├── 매출전표_오류포함.xlsx        # 의도적 오류 데이터
└── 매입전표_혼합형식.xlsx        # 한영 혼용
```

### 데모 순서

#### Step 1: 깔끔한 데이터 (기본)
1. `AI파싱` 메뉴 클릭
2. `sample_data/매출전표_202501.xlsx` 업로드
3. **결과**: 품질 점수 A (90점 이상), 모든 컬럼 100% 매핑
4. **멘트**: "이렇게 정리된 데이터는 당연히 잘 됩니다."

#### Step 2: 더존 스타일 (컬럼명 다름)
1. `sample_data/messy_samples/매출전표_더존스타일.xlsx` 업로드
2. **포인트**:
   - `매출일자` → `전표일자` 자동 매핑
   - `거래선명` → `거래처명` 자동 매핑
   - `원화매출` → `원화환산액` 자동 매핑
3. **멘트**: "컬럼명이 달라도 AI가 의미를 파악해서 자동 매핑합니다."

#### Step 3: SAP 스타일 (영문)
1. `sample_data/messy_samples/매출전표_SAP스타일.xlsx` 업로드
2. **포인트**:
   - `BUDAT` → `전표일자` 자동 매핑
   - `KUNNR` → `거래처코드` 자동 매핑
   - `NETWR` → `공급가액` 자동 매핑
   - 유럽식 날짜 형식 (DD.MM.YYYY) 자동 인식
3. **멘트**: "SAP, Oracle 등 해외 ERP 형식도 AI가 자동 인식합니다."

#### Step 4: 오류 데이터 (핵심 차별화!)
1. `sample_data/messy_samples/매출전표_오류포함.xlsx` 업로드
2. **포인트**:
   - 음수 수량 감지 (행 5)
   - 0 단가 감지 (행 12)
   - 음수 금액 감지 (행 18)
   - 비정상적으로 큰 값 감지 (행 40)
   - 품질 점수 하락 (D등급)
3. **멘트**: "이상치도 자동 감지합니다. 어떤 행에 문제가 있는지까지 알려드립니다."

## 데모 시나리오 2: ERP 데이터 → AI 손익계산서

### 목표
- 복잡한 ERP 원천 데이터를 AI가 조합하여 손익계산서 자동 생성

### 샘플 파일 위치
```
sample_data/
├── 매출전표_202501.xlsx      # 252건, ~300억원
├── 매입전표_202501.xlsx      # 73건, ~250억원
├── 급여대장_202501.xlsx      # 125명, ~5.4억원
├── 제조경비_202501.xlsx      # 25건, ~4.9억원
├── 재고현황_202501.xlsx      # 19건
└── 판매관리비_202501.xlsx    # 35건, ~4.2억원
```

### 데모 순서

1. `데이터 연동` 메뉴 클릭
2. 매출전표, 매입전표 업로드 (필수)
3. 추가로 급여대장, 제조경비, 재고현황, 판매관리비 업로드 (선택)
4. "AI 손익계산서 생성" 클릭
5. **결과**:
   - 손익계산서 자동 생성
   - 매출총이익률, 영업이익률 자동 계산
   - AI 분석 코멘트 생성

### 멘트
> "ERP에서 뽑은 원천 데이터만 넣으면,
> AI가 알아서 조합해서 손익계산서를 만들어 드립니다.
> 더 이상 수작업으로 정리할 필요 없습니다."

## 데모 시나리오 3: 무역 서류 OCR

### 목표
- PDF/이미지 무역 서류를 AI가 자동 파싱

### 샘플 파일 (준비 필요)
- Commercial Invoice PDF
- Bill of Lading PDF

### 데모 순서
1. `서류` 메뉴 클릭
2. Invoice 업로드
3. **결과**: 인보이스 번호, 바이어명, 품목, 금액 자동 추출
4. B/L 업로드
5. "서류 대사" 클릭
6. **결과**: Invoice vs B/L 수량/금액 자동 비교, 불일치 하이라이트

## 테스트 체크리스트

### 데모 전 확인사항
- [ ] 백엔드 서버 실행 (`python -m uvicorn backend.main:app --reload`)
- [ ] 프론트엔드 서버 실행 (`cd frontend && npm run dev`)
- [ ] 샘플 파일 준비 확인 (`sample_data/` 폴더)

### API 엔드포인트
- `POST /api/smart-parser/analyze` - 스마트 파싱
- `POST /api/erp-sync/upload` - ERP 데이터 업로드
- `POST /api/erp-sync/session/{id}/generate` - 손익계산서 생성
- `POST /api/documents/upload/invoice` - 인보이스 OCR

## Q&A 예상 질문

### Q: 어떤 ERP와 연동되나요?
**A**: 현재는 엑셀 업로드 방식입니다. 더존, SAP, Oracle 등 어떤 ERP에서 뽑은 데이터든 AI가 자동 인식합니다. 향후 API 직접 연동도 가능합니다.

### Q: 컬럼명이 완전히 다르면 어떻게 되나요?
**A**: AI가 샘플 데이터를 보고 의미를 파악합니다. 예를 들어 "매출일자", "판매일", "Sale Date" 모두 같은 의미로 인식합니다.

### Q: 오류 데이터는 어떻게 처리되나요?
**A**: 자동 감지 후 경고를 표시합니다. 어떤 행에 어떤 문제가 있는지 상세히 알려드리고, 수정 후 재업로드하시면 됩니다.

### Q: 보안은 어떻게 되나요?
**A**: 업로드된 데이터는 분석 후 즉시 삭제됩니다. 클라우드 저장 없이 실시간 처리 방식입니다.
